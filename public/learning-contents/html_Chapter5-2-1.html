📦 div와 시맨틱 태그 비교

그냥 <div>만 써도 화면은 똑같이 나오는데...
왜 굳이 시맨틱 태그를 써야 할까? 🧐

<div>는 아무 의미 없는 '상자'예요.
반면 시맨틱 태그는 '이름표가 붙은 상자'라서 컴퓨터가 이해하기 훨씬 쉽답니다.

1. 코드로 비교해보기

두 코드는 화면에 똑같이 보이지만, 의미는 완전히 달라요.

div만 사용 (의미 없음) 
<div class="header">로고</div>
<div class="nav">메뉴</div>
<div class="main">본문</div>
<div class="footer">바닥글</div>

컴퓨터: "전부 다 div(상자)네? 뭐가 중요한 거지?" 🤷‍♂️

시맨틱 태그 사용 (의미 있음) 
<header>로고</header>
<nav>메뉴</nav>
<main>본문</main>
<footer>바닥글</footer>

컴퓨터: "아하! 여기가 헤더고, 여기가 본문이구나!" 💡

2. 왜 시맨틱 태그를 써야 할까요?

❌ <div>의 단점

  • 코드를 읽어도 무슨 역할인지 알기 힘들다.
  • 검색 엔진(구글)이 중요한 내용을 못 찾는다.
  • 시각장애인용 리더기가 구조를 파악하기 어렵다.

✅ 시맨틱 태그의 장점

  • 태그 이름만 봐도 역할을 바로 알 수 있다. (가독성 UP)
  • 검색 엔진 최적화(SEO)에 유리하다.
  • 웹 접근성(장애인 지원)이 좋아진다.

3. 그럼 <div>는 언제 쓰나요?

시맨틱 태그로 표현하기 애매한 단순한 그룹 묶기나 디자인용 컨테이너가 필요할 때 씁니다.

<div class="card-wrapper"> (단순 묶음용)
<article> (실제 내용)

내용...

  • CSS 스타일을 적용하기 위해 묶을 때
  • 특별한 의미 없이 레이아웃을 잡을 때
핵심 정리

  • 🥈
    div는 보조적으로!
